About Me

I'm a first-year M.S.–Ph.D. student at Korea University. I work on mobile sensing and sequential learning systems, with recent projects in IMU-based pedestrian dead reckoning (PDR), robust signal processing, and real-time deployment on Android.

I enjoy building end-to-end pipelines (data collection → modeling → evaluation → deployment) and making results reproducible with clean code and logging. I'm broadly interested in time-series modeling, uncertainty-aware learning, and efficient adaptation for real-world perception and decision-making.

Research Projects

Indoor Pedestrian Dead Reckoning (PDR) System

2024 – Present

Problem: Existing PDR methods suffer from drift and fail to adapt to diverse phone-holding modes in GPS-denied indoor environments.

Approach: Built adaptive 3-mode step detection (OnHand/InPocket/HandSwing) with peak-valley-zero-crossing state machine, quaternion transforms, and C++17/JNI engine. Compared 5 stride formulas and 7 step detection methods on 700+ datasets.

Result: 2 Android apps, 42 Python analysis tools, and a C++ engine (7,400+ LOC).

Kotlin C++17 Python NDK/JNI FFT Quaternion
[GitHub]

Deep RL-based PDR Step Length Personalization

2025 – Present

Problem: Fixed stride models cause systematic drift because gait characteristics vary across individuals.

Approach: Designed a custom Gymnasium environment (38-dim state, 41 actions) to learn per-user scaling factors via DQN/PPO with Zeromap collision detection.

Result: Implemented environment and training pipeline; project paused at preliminary experimentation stage.

DQN PPO stable-baselines3 Gymnasium pybind11

Multi-Building Indoor Navigation Platform

2024 – Present

Problem: No unified system for multi-building, multi-floor indoor routing with real-time floor detection.

Approach: Built ZeroMap apps (compass + PDR), barometer-based floor detection (Weighted Scoring FSM), and C++ simulation framework (38K LOC) with OpenCV visualization.

Result: Tested and demonstrated across 4 buildings (KU campus, Daewoo apartment, Suwon station, naval vessel) with 3 simulators and Express.js / Spring Boot backends.

Android C++17 OpenCV Express.js Spring Boot Unity 3D

Real-time Indoor Object Detection & OCR

2025 – Present

Problem: Indoor navigation lacks visual context—users cannot automatically identify stairs, exits, or elevators from camera feed.

Approach: Trained YOLOv8n (3.2M params, 8 classes) and deployed on Android via ONNX Runtime + Camera2. Integrated OCR for sign text extraction.

Result: 90%+ precision (stair mAP50: 84.5%, exit sign: 99.5%) with real-time mobile inference at 640×640.

YOLOv8 ONNX Runtime Kotlin Camera2 API
[GitHub]

Automated Floor Plan Processing & Indoor Map Generation

2024 – Present

Problem: Converting CAD drawings (DXF) into indoor map data is manual and doesn't scale across buildings.
