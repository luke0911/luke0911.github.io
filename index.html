About Me

I am a M.S. student at Korea University, researching indoor positioning and pedestrian navigation. I develop PDR (Pedestrian Dead Reckoning) systems that estimate indoor positions using only smartphone IMU sensors without GPS, and extend my research to deep reinforcement learning for gait personalization and computer vision-based indoor object recognition.

I design and implement end-to-end systems across Android/Kotlin, C++, and Python — from sensor signal processing and deep learning model training to mobile app deployment.

Graduate Research Projects

Indoor Pedestrian Dead Reckoning (PDR) System

2025 – Present

A comprehensive PDR research platform with adaptive step detection across 3 phone-holding modes (OnHand / InPocket / HandSwing), a C++17 native engine, and 42 Python analysis tools. Compared 5 stride estimation formulas and 7 step detection methods using 700+ sensor datasets.

2 Android apps · 42 Python tools · C++ 7,400+ LOC · 700+ sensor datasets
Kotlin C++17 Python NDK/JNI FFT Quaternion

Deep RL-based PDR Step Length Personalization

2025 – Present

A system that automatically learns optimal per-user step length scale factors (Alpha) using DQN/PPO. Designed a custom Gymnasium environment with 38-dim state space (22 IMU features + 16 statistics) and 41 discrete actions. Trained with Zeromap indoor collision detection.

38-dim state space · 500K+ training timesteps · 22-floor map data
DQN PPO stable-baselines3 Gymnasium pybind11

Multi-Building Indoor Navigation Platform

2025 – Present

Indoor navigation system supporting KU campus, Daewoo apartment, Anam hospital, and Suwon station. Built ZeroMap apps, a C++ simulation framework (38K LOC), and Express/Spring Boot backends. Developed IMU-based floor detection (Weighted Scoring FSM) and dual-pipeline PDR comparator.

4 buildings · C++ 38,000+ LOC · 3 simulators
Android C++17 OpenCV Express.js Spring Boot Unity 3D
[KU-LAB] [Indoor-Navigation-Sim]
