Indoor Pedestrian Dead Reckoning (PDR) System

2024 – Present

Problem: GPS-denied indoor environments lack reliable pedestrian tracking; existing PDR methods suffer from drift and fail to adapt to diverse phone-holding modes.

Approach: Designed an adaptive 3-mode step detection engine (OnHand/InPocket/HandSwing) using a peak-valley-zero-crossing state machine, quaternion-based coordinate transforms, and a C++17 native library bridged via JNI. Compared 5 stride estimation formulas and 7 step detection methods on 700+ sensor datasets from 8 participants.

Result: Built 2 Android apps, 42 Python analysis tools, and a C++ engine (7,400+ LOC). Best step detection achieved 97% recall with 1% false positive; best stride formula (Tian) reached 3–6% error.

Kotlin C++17 Python NDK/JNI FFT Quaternion

Deep RL-based PDR Step Length Personalization

2025 – Present

Problem: Fixed stride estimation models introduce systematic drift for different users because gait characteristics vary significantly across individuals.

Approach: Proposed learning a per-user scaling factor (Alpha) instead of directly estimating stride. Designed a custom Gymnasium environment with 38-dim state space (22 IMU features + 16 statistics), 41 discrete actions, and a composite reward (goal-reaching + collision avoidance + distance matching). Trained DQN/PPO agents over 500K+ timesteps with Zeromap indoor collision detection.

Result: Demonstrated automatic per-user gait adaptation with 22-floor map validation. The personalization approach reduced position drift compared to fixed-parameter baselines.

DQN PPO stable-baselines3 Gymnasium pybind11

Real-time Indoor Object Detection & OCR

2025 – Present

Problem: Indoor navigation systems lack visual context; users cannot automatically identify stairs, exits, or elevators from camera feed for seamless wayfinding.

Approach: Trained a YOLOv8n model (3.2M params) on 8 indoor object classes and deployed it on Android via ONNX Runtime with Camera2 real-time inference. Integrated OCR for sign text extraction.

Result: Achieved 90%+ precision on key classes (stair mAP50: 84.5%, exit sign: 99.5%) with real-time mobile inference at 640×640 resolution.
